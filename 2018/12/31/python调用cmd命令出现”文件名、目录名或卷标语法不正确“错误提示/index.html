python调用cmd命令出现“文件名、目录名或卷标语法不正确”错误提示

发表于 2018-12-31

今天在Pycharm中需要执行cmd命令时出现了这个问题，需要调用一个绝对路径下的可执行文件，
` python

Dnconsole.console = D:\dnplayer2\dnconsole.exe

cmd = os.popen(Dnconsole.console+” list2”)
text = cmd.read()

#print("text: ",text)
cmd.close()

‘’’

找了好久一直以为是反斜杠的问题，最后终于找到了坑！我需要在命令行中执行一个可执行文件，写的的绝对路径,而这个绝对路径这么长我直接右键–安全–复制的，很简单美滋滋对不对，然而呀，复制完的头上有个隐藏符号，这个Unicode 202a字符，这个字符的含义是left to right，也就是从左到右的阅读顺序，在写代码的时候显示不出来，但是在命令行中可以显示出来，解决方法就是：把鼠标放到D前面，然后backspace向左删除一个字符，会发现显示上没有发生变化，但是控制字符已经被删除了。
看网上的帖子：https://blog.csdn.net/york1996/article/details/81871994
