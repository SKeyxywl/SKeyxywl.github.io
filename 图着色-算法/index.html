图着色|算法|

问题描述

图着色问题（Graph Coloring Problem, GCP）　又称着色问题，是最著名的NP-完全问题之一。
数学定义：给定一个无向图G=（V, E），其中V为顶点集合，E为边集合，图着色问题即为将V分为K个颜色组，每个组形成一个独立集，即其中没有相邻的顶点。其优化版本是希望获得最小的K值。
图的m-着色判定问题——给定无向连通图G和m种不同的颜色。用这些颜色为图G的各顶点着色，每个顶点着一种颜色，是否有一种着色法使G中任意相邻的2个顶点着不同颜色?

主要思路

这个问题就像八皇后之类的题目类似，枚举+回溯，具体点就是每个点都把n个颜色放一遍，每当一个点放了一种颜色，就开始填下一个点，等这一条路到头之后再回到这个点并填入下一个颜色继续往下走。 注意：有些算法中用回溯的时候要保证每一次回到当前这个点开始下个路线时“状态”要保持一致

#输入说明：本例子为最简单的三个颜色示例，第一行一个n，接下来的n行代表各个点之间的连接状态
示例1：
5
1 2
1 3
1 4
2 3
4 3

‘’’java
import java.util.Scanner;

public class map {
private static int e[][];//存储顶点连接情况的二维矩阵
private static int sear[];//存储每个顶点的染色情况
private static int colornum =3;
private static int SUM =0;

//递归函数
private static void fun(int index){
    //进来的时候这个点已经染好色，判断这个点可以染这个色吗
    if(isOk(index)){
        //到这说明这个点的当前颜色成立，接着判断是不是最后一个点，不是的话继续把下一个点染色，
        //然后进行递归
        if(index==sear.length-1){
            SUM++;
            System.out.println("第"+SUM+"种方案：");
            show(index);
        }else{
            for(int i=1;i<=colornum;i++){
                sear[index+1]=i;
                fun(index+1);
            }
        }
    }
}

private static void show(int index) {
    for(int i=1;i<sear.length;i++){
        System.out.println(i+"点的颜色是："+sear[i]);
    }
}

private static boolean isOk(int index) {
    for(int i=1;i<index;i++){
        if(e[index][i]==1&&sear[index]==sear[i]){//如果和它相连的点的颜色相同
            return false;
        }
    }
    return true;
}
public static void main(String args[]){
    Scanner scanner = new Scanner(System.in);
    int n = scanner.nextInt();
    e = new int[n][n];
    sear = new int[n];
    for(int i=0;i<n;i++){
        int a =scanner.nextInt();
        int b =scanner.nextInt();
        e[a][b]=1;
        e[b][a]=1;
    }
    fun(0);
}

}
